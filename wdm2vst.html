插件一览

套件包含 4 个 VST3 插件，覆盖音频路由的所有场景

WDM2VST

核心 · 输入

将 Windows WDM 虚拟音频设备的音频捕获到 DAW 中。支持 IPC 零延迟模式和 WASAPI 录音模式，可捕获任意系统声音。

系统 → DAW

VST2WDM

核心 · 输出

将 DAW 音频发送到 WDM 物理/虚拟设备。支持选择输出设备、音量控制、限制器，可实现 DAW 到系统的音频路由。

DAW → 设备

Audio Send

扩展 · 发送

跨插件音频路由发送端。支持 IPC 本机通信、LAN 局域网传输、手机监听。可广播到多个接收端。

插件 → 网络

Audio Receive

扩展 · 接收

跨插件音频路由接收端。接收来自 Audio Send 的音频流，支持本机 IPC 零延迟和 LAN 远程接收。

网络 → 插件

使用方法

从安装到使用，5 分钟上手

版本更新检测

内置自动化版本侦测系统。当检测到新版本时，UI自动变形展现极客风格呼吸幻彩按钮，全自动重定向至最新下载地址。

内核级遥测监控 (Telemetry)

首创诊断式宿主音频配置收集，实时协助排查因未知原因引发的爆音问题。

安装插件

运行 WDM2VST_Public_v3.2.0_Setup.exe，插件自动安装到 C:\Program Files\Common Files\VST3

打开 DAW

打开 Studio One / Cubase / Ableton / FL Studio 等 DAW，扫描 VST3 插件目录

创建轨道并加载插件

新建音频轨道，在效果器插槽中加载 WDM2VST 或 VST2WDM

选择音频设备

在插件界面下拉列表中选择目标设备。IPC 模式选虚拟设备，WASAPI 模式选物理设备

开始录制/播放

点击「启动」按钮，音频流开始传输。DAW 中即可看到电平变化

💡 IPC 零延迟模式

如果安装了 WDM2VST 虚拟音频驱动，推荐使用 IPC 模式。通过共享内存实现真正的零延迟传输，音质无损且 CPU 占用极低。

⚠️ 采样率匹配

确保 DAW 采样率与系统音频设备一致（48000Hz 或 44100Hz），否则可能出现音高异常或噪音。

O Deus 噪声来源分析

  • Ring Buffer 同步问题 — 读写指针竞争导致读到部分旧数据
  • 插值 / 重采样 — 可能在读取 IPC 数据后做了采样率转换
  • 缓冲区溢出/欠载处理 — 缓冲跟不上时填充了非零数据
  • 格式转换 — 传输中可能经历了 int→float 转换
我们的实现使用 IpcAudioBuffer 结构直接在内核共享内存中传递 float32 样本，没有任何格式转换或插值，因此实现了 Bit-Perfect 传输。

常见问题

DAW 扫描不到插件？

确认安装目录为 C:\Program Files\Common Files\VST3，在 DAW 中添加该路径并重新扫描。

IPC 模式无法连接？

需要先安装 WDM2VST 虚拟音频驱动。没有驱动请使用 WASAPI 模式。

有杂音或音高不对？

检查采样率是否匹配：DAW 和系统设备须一致，建议统一 48000Hz。

LAN 模式连不上？

点击 Audio Send 的「修复防火墙」按钮，确保两台电脑在同一子网。
