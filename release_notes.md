## 🚀 WDM2VST V3.2.1 稳定版 正式发布 

### 🎵 核心音频与引擎修复 (Core Audio & Engine)
- **彻底屏蔽双重播放 Bug**：完美修复在 DAW 加载 VST2WDM 插件并启用 IPC 时，底层物理设备依然会并发播放从而产生幽灵重音的历史遗留故障（通过重构端点释放机制实现）。
- **内核级 DPF 状态机诊断 (Kernel Diagnostics)**：底层 WaveRT 音频流引擎全新注入无损代码日志队列，精准捕捉 DAW 与系统的采样率协商断层，通过 DebugView 实时追踪，深度剖析并终结“采样率不匹配导致的各类鬼畜爆音”。

### 📦 安装包与架构全面重构 (Installer & Architecture)
- **安装目录 100% 纯净洗地 ({tmp} Clean-Up)**：重写 Inno Setup 安装向导内核，引入特殊的动态缓存提取机制。所有注册表导入、批处理配置等冗余工具文件会在驱动成功注册的瞬间自动彻底自毁，还你一个绝对洁癖级的 Program Files 驱动环境。
- **全套工程构建链重装上阵**：剔除了之前套娃式的臃肿 CMake 冗余目录层，全维清剿因 MSBuild 散落遗留污染所引发的嵌套交叉崩盘故障；现在直接产出轻量、纯净的标准 VST3 格式。
- **全版本矩阵大一统 (Ver 3.2.1)**：Driver 驱动、Plugins 插件层与 Installer 安装器三核心端点版本编号迎来历史性的一致统配，彻底结束自发版以来的版本碎片化混乱。

### 🛡️ 体验守护与避坑指南 (Troubleshooting UX)
- **Code 37 阻断拦截引导**：在安装向导中原生集成微软 HVCI（内核隔离/内存完整性）的提示。针对报错代码 37 / 52（由于系统严格数字签名保护导致未能初始化的虚拟硬件），提供可视化的极速图形避障通路引导。
  > *解决此问题，仅需关闭 Windows 安全中心 -> 设备安全性 -> 内核隔离中的【内存完整性 (Memory Integrity)】后重启即秒刻恢复正常。*
