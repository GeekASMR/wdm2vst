Virtual Audio Router

VST3 Plugin Suite — User Manual
Version 2.1.0

1. Product Overview

Virtual Audio Router is a Windows virtual audio driver + VST3 plugin suite that enables zero-latency audio routing between:

  • System audio and your DAW (Digital Audio Workstation)
  • DAW audio to your phone for mobile monitoring
  • DAW audio between multiple PCs on the same LAN

3. Installation

3.1 Driver

  1. Run VirtualAudioRouter_Setup.exe as Administrator
  2. Accept the driver signing prompt
  3. Restart your computer
  4. Verify in Sound Settings: Virtual 1/2 ~ 7/8 (playback) and Mic 1/2 ~ 7/8 (recording)

3.2 Plugins

Copy .vst3 folders to:

C:\Program Files\Common Files\VST3\

Then rescan plugins in your DAW.

4. Plugin Usage Guide

Figure 2: Four VST3 Plugin Interfaces

4.1 WDM2VST — System Audio → DAW

Purpose: Capture any system audio (music, browser, game, etc.) into a DAW track.

Controls

ControlDescription
DEVICESelect audio input source
LEVELAdjust input volume (0 – 200%)
LimiterSoft limiter to prevent clipping
Latency ModeSafe (8ms+) / Fast (4ms+) / Extreme (2ms+) / Insane (1ms+)
CN/ENChinese / English language toggle

Connection Modes

ModeSelectionLatency
Standard RecordingAny microphone/line inputHardware dependent
System LoopbackLoopback / Speaker device~20ms
IPC ChannelVirtual 1/2 [IPC] etc.~0ms (zero latency)
Tip: IPC mode provides zero-latency routing. Select the IPC channel matching the Virtual Speaker endpoint used by your system audio.

Quick Start: Record System Audio

  1. Set Windows default playback device to Virtual 1/2
  2. Insert WDM2VST on a DAW track
  3. Select Virtual 1/2 [IPC] from the device dropdown
  4. Play audio on your PC — it appears on the DAW track

4.2 VST2WDM — DAW → System Audio + Mobile

Purpose: Route DAW audio to physical speakers/headphones and optionally to phone monitoring.

Controls

ControlDescription
DEVICESelect audio output device
LEVELOutput volume (0 – 200%)
LimiterSoft limiter
MobileEnable phone monitoring via WebSocket
WS PortWebSocket port (default: 18820)
QRShow QR code for phone connection
Fix FWAuto-configure Windows Firewall

Mobile Monitoring Steps

  1. Enable the Mobile toggle
  2. Click Fix FW (first time only)
  3. Click QR — scan with your phone camera
  4. Audio plays in phone browser in real-time
Note: Phone and PC must be on the same WiFi network.

4.3 Audio Send — DAW → Network

Purpose: Stream DAW audio to other PCs or mobile devices.

ControlDescription
CHANNELP2P channel 1–8 (must match receiver)
IPC/LAN/MobileEnable transport modes
LAN TARGET127.0.0.1 (same PC) / 192.168.x.x (other PC) / .255 (broadcast)
UDP/WS PORTNetwork ports
START/STOPStart or stop sending

4.4 Audio Receive — Network → DAW

Purpose: Receive audio from Audio Send on this or another PC.

ControlDescription
CHANNELMust match sender’s channel
IPC/LANEnable transport modes
UDP PORTMust match sender’s port
START/STOPStart or stop receiving

5. Common Scenarios

5.1 Record System Audio

[Windows Audio] → Virtual 1/2 → IPC → WDM2VST → DAW Track

5.2 Monitor DAW on Phone

[DAW Master] → VST2WDM (Mobile ON) → WebSocket → Phone Browser

5.3 Stream to Another PC

[DAW-A] → Audio Send (LAN) → UDP → Audio Receive → [DAW-B]

6. Troubleshooting

ProblemSolution
No virtual audio devicesReinstall driver, check Device Manager
IPC channel no audioVerify plugin and driver use same channel ID
Mobile QR scan failsClick Fix FW, check same WiFi network
LAN no audio receivedCheck firewall, verify IP and port match
Audio cracklingIncrease DAW buffer size or use Safe latency mode

7. Language Support

All plugins support Chinese / English toggle via the CN/EN button (top-right corner). Default: Chinese.
