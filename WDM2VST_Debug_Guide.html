📝 WDM2VST 驱动故障排查与日志收集指南

当虚拟音频驱动（虚拟声卡）出现声音卡顿、跳音、无法出声或设备被占用等深层问题时，常规截图无法判断原因。请按照以下步骤截取内核级日志文件供开发人员分析。

🔔 步骤 0：解除 Windows 拦截 (仅初次需要)

由于最近的 Windows 系统默认禁用了驱动日志输出，我们需要先执行一次注册表解除锁定：

  1. 按键盘 Win 键，搜索 "命令提示符" 或 "CMD"。
  2. 点击右侧的 "以管理员身份运行"。
  3. 复制以下内容，右键粘贴进去并回车：
    reg add "HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Debug Print Filter" /v "DEFAULT" /t REG_DWORD /d 0xFFFFFFFF /f
  4. 提示“操作成功完成”后，重启一次电脑！。

🛠️ 步骤 1：下载并打开日志抓取工具 (DebugView)

我们使用微软官方的系统排查内核工具 DebugView。

  • 如果您尚未下载，请点此 下载 DebugView 官方绿色版
  • 解压下载的压缩包。
  • 找到里面的 dbgview64.exe，必须鼠标右键 -> 选择“以管理员身份运行”。

🎯 步骤 2：监听内核捕捉

打开 DebugView 软件后，为了只看到我们的虚拟声卡的日志：

  1. 点击屏幕最上方菜单栏的 Capture。
  2. 确保 Capture Kernel (捕捉内核) 是打勾状态的。
  3. （可选）点击菜单栏的漏斗图标（或者按 Ctrl+L），在 "Include" 输入框中输入 *WDM2VST*，然后点 OK。这样屏幕就不会被别的系统乱码刷屏了。

🔥 步骤 3：复现问题

这时候软件会在后台一直监控。请开启您的音乐软件或 DAW (比如 Studio One / OBS)：

  • 像平常使用一样去复现你的 BUG（比如直到听到破音、杂音发生）。
  • 复现完成后，看一眼 DebugView 窗口，里面应该刷出了密密麻麻的 [WDM2VST STREAM...] 开头的英文。

📁 步骤 4：保存并发送给开发人员

  1. 在 DebugView 里，点击左上角的 File -> Save As...。
  2. 把文件保存为 虚拟声卡故障记录.log。
  3. 关掉 DebugView， 把这个文件发给我们！
