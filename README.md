📅 2026-03-20 · 🔧 v2.1.0 · 🎵 4 个 VST3 插件 · 💻 Windows 64-bit


### ⚠️ 常见安装问题排查 (Troubleshooting)
- **设备管理器报错 Code 37 或 Code 52（无法初始化硬件 / 数字签名验证失败）**：
  这通常是因为较新的 Windows 10/11 系统默认开启了严格的内核隔离。**解决方法**：
  1. 打开 Windows 安全中心 -> 设备安全性 -> 内核隔离。
  2. 关闭【**内存完整性 (Memory Integrity)**】。
  3. 重启电脑即可恢复正常工作状态。

## 🎯 项目简介

WDM2VST 是一套 **虚拟音频路由插件**，安装在你的 DAW（数字音频工作站）中，
可以在 **Windows 系统音频** 和 **DAW 内部** 之间自由传输音频信号。
无需额外硬件，无需复杂布线，一键实现系统声音的专业级录制和处理。

```
🖥️ Windows 系统声音 ⟷ 🔌 WDM 虚拟驱动 ⟷ 🔗 IPC 共享内存 ⟷ 🎛️ DAW 插件
```
> 整条链路 Bit-Perfect（比特级完美）无损传输 · 零格式转换 · 超低延迟

## 🎙️ 插件一：WDM2VST — 系统声音捕获到 DAW

| 属性 | 说明 |
|------|------|
| 功能定位 | 将 Windows 系统播放的声音 **实时捕获** 到 DAW 轨道中 |
| 信号方向 | WDM 驱动 → DAW（系统声音 → DAW 输入） |
| 典型场景 | • 直播时在 DAW 中混合系统声音和麦克风<br>• 录制游戏/视频通话/浏览器音频<br>• 对系统声音做实时 EQ、压缩等后期处理<br>• 多通道系统声音同时捕获（支持 4 对立体声通道） |
| 传输方式 | IPC（进程间通信）共享内存，**Bit-Perfect 无损** |
| 延迟模式 | Extreme（极速 2ms+）/ Fast（快速 4ms+）/ Safe（安全 8ms+） |
| 特色功能 | 音量控制、Limiter（限制器）、实时电平表、延迟显示 |

## 📐 技术分析

### 传输架构（两者相同）

```
驱动渲染端点 → 内核共享内存 (IPC/LPC) → VST 插件读取
```

### 为什么同为 IPC（进程间通信），我们更优？

| 方面 | 我们的 WDM2VST | O Deus wdm2vst |
|------|---------------|----------------|
| IPC（进程间通信）方式 | Shared Memory（共享内存） | Shared Memory（共享内存） |
| Data Format（数据格式） | float32（32位浮点）直通 | float32（32位浮点） |
| Signal Quality（信号质量） | Bit-perfect（比特级完美，0% THD） | 有噪声（0.06% THD） |
| SNR（信噪比） | 128 dB | 58-61 dB |

### O Deus 噪声来源分析

- **Ring Buffer（环形缓冲区）同步问题** — Read/Write Pointer（读写指针）竞争导致读到部分旧数据或中间状态
- **Interpolation（插值）/ Resampling（重采样）** — 可能在读取 IPC 数据后做了 Sample Rate Conversion（采样率转换）
- **Buffer Overflow/Underrun（缓冲区溢出/欠载）处理** — 当缓冲区跟不上时填充了非零数据
- **Format Conversion（数据格式转换）** — 可能在 IPC 传输过程中经历了格式转换（如 int→float 整数转浮点）

### 我们的优势

> 我们的实现使用简洁的 `IpcAudioBuffer`（IPC音频缓冲区）结构直接在 Kernel Shared Memory（内核共享内存）中传递 float32（32位浮点）样本，没有经过任何 Format Conversion（格式转换）或 Interpolation（插值），因此实现了 **Bit-Perfect（比特级完美）** 传输。
